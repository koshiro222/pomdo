# Issue作成ガイドライン

## Issueテンプレート

```
[種別] 簡潔なタイトル

## 概要
何を実装/修正/改善するかを1〜2文で説明

## 詳細
実装内容の詳細説明
- 具体的な機能仕様
- 変更の影響範囲
- 実装方法のポイント

## TODO
- [ ] タスク1
- [ ] タスク2
```

## 種別一覧

| 種別 | 用途 |
|------|------|
| `[feat]` | 新機能の追加 |
| `[fix]` | バグ修正 |
| `[refactor]` | リファクタリング |
| `[docs]` | ドキュメント更新 |
| `[test]` | テスト関連 |
| `[chore]` | その他の作業 |

## TODOに必ず含める項目

作業開始時と完了時に実施するタスクを必ず含めてください：

- [ ] 専用featureブランチを作成する
- [ ] 実装内容のテスト・動作確認
- [ ] コミットする
- [ ] リモートブランチにpush
- [ ] PRを作成する（まだ作成されていない場合）
