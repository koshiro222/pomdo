1. インポート追加：

2. bgmRouterにcreate mutation追加（getAllの後）：
```typescript
create: adminProcedure
  .input(createBgmTrackSchema)
  .mutation(async ({ ctx, input }) => {
    const { db, schema } = ctx

    // ファイルサイズチェック（Base64デコード後）
    const buffer = Buffer.from(input.fileBase64, 'base64')
    const MAX_FILE_SIZE = 10 * 1024 * 1024 // 10MB

    if (buffer.byteLength > MAX_FILE_SIZE) {
      throw new TRPCError({
        code: 'BAD_REQUEST',
        message: 'ファイルサイズは10MB以下にしてください',
      })
    }

    // ファイル名生成（UUID）
    const filename = `${crypto.randomUUID()}.mp3`

    // R2にアップロード
    await ctx.env.BGM_BUCKET.put(filename, buffer, {
      httpMetadata: { contentType: 'audio/mpeg' },
    })

    // DBに登録
    const [created] = await db
      .insert(schema.bgmTracks)
      .values({
        title: input.title,
        artist: input.artist,
        color: input.color,
        filename,
        tier: input.tier ?? 'free',
      })
      .returning()

    return created
  })
```

3. Context型拡張（ファイル先頭）：
```typescript
interface BgmContext {
  user: SessionUser | null
  db: any
  schema: any
  env: {

```

実装の注意点：
- `ctx.env`へのアクセスのため、Context型を拡張またはany型を使用
- R2操作はEdge Runtime標準APIを使用（Buffer.from, crypto.randomUUID）
- エラーメッセージは日本語（CONTEXT.md決定）
- tierのデフォルト値は"free"（CONTEXT.md決定）
- ファイルサイズはデコード後のバイト数でチェック（Base64エンコードにより約33%増加）

todos.tsのcreateパターンを参考に、db.insert().values().returning()のチェーンを使用。
